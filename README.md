# PersonalBlog

Backend API for a personal blog project built with ASP.NET Core, Entity Framework Core, and PostgreSQL.

## Requirements

- .NET SDK installed
- PostgreSQL running locally
- A PostgreSQL database created for the project

## Local setup

Clone the repository and restore the dependencies:

```bash
git clone https://github.com/frezatti/PersonalBlog.git
cd PersonalBlog
dotnet restore
```

Database configuration

The project uses PostgreSQL. Configure the connection string with user secrets instead of committing real credentials to GitHub.

From the folder that contains PersonalBlog.csproj, run:

dotnet user-secrets set "ConnectionStrings:DefaultConnection" "Host=localhost;Port=5432;Database=personal_blog_db;Username=postgres;Password=YOUR_POSTGRES_PASSWORD"

Replace YOUR_POSTGRES_PASSWORD with your local PostgreSQL password.

Then apply the migrations:

dotnet ef database update
JWT setup

This project uses JWT Bearer Authentication.

A JWT key is required locally so the API can create and validate login tokens. This key is used only by the API server. Normal users do not need to know it.

Each person running the project locally should generate their own JWT key.

Generate a key:

openssl rand -base64 32

Then add it to user secrets:

dotnet user-secrets set "Jwt:Key" "PASTE_GENERATED_KEY_HERE"
dotnet user-secrets set "Jwt:Issuer" "PersonalBlog"
dotnet user-secrets set "Jwt:Audience" "PersonalBlog"

Replace PASTE_GENERATED_KEY_HERE with the value generated by openssl rand -base64 32.

Do not commit the real JWT key to GitHub.

Running the API

From the repository folder, run:

dotnet run --project PersonalBlog

When running in development, Swagger should be available at:

https://localhost:PORT/swagger

The exact port depends on the local launch profile.

Authentication flow

To test protected endpoints in Swagger or Postman:

Create a user.
Log in with that user.
Copy the returned JWT token.
Send the token as a Bearer token in the Authorization header.

Header format:

Authorization: Bearer YOUR_TOKEN_HERE
